Git

Git 是一个非常流行的版本控制系统，常用命令主要分为以下几类：基本操作、分支管理、查看状态和日志、远程仓库管理等。下面是一些常用的 Git 命令及其简要介绍：

1. 基本操作

  • git init：初始化一个新的 Git 仓库。通常在新项目的根目录下运行这个命令。
  • git clone <repository>：克隆一个远程仓库到本地。
  • git add <file>：将文件添加到暂存区，准备提交。可以用 . 来添加所有更改的文件。
  • git commit -m "<message>"：提交暂存区的文件到本地仓库，并附上提交信息。
  • git status：查看当前仓库的状态，包括哪些文件被修改、哪些文件在暂存区等。
  • git diff：显示未暂存的文件的更改内容。也可以用 git diff --staged 查看已暂存但未提交的更改。
  • git log：显示提交历史记录。可以用 git log --oneline 查看简洁的提交历史。

2. 分支管理

  • git branch：列出所有本地分支。如果添加分支名，可以创建一个新分支。
  • git checkout <branch>：切换到指定的分支。
  • git checkout -b <branch>：创建并切换到新分支。
  • git merge <branch>：将指定分支合并到当前分支。
  • git branch -d <branch>：删除本地分支。

3. 远程仓库管理

  • git remote add origin <url>：添加远程仓库地址。通常用于将本地仓库连接到远程仓库。
  • git remote -v：查看当前配置的远程仓库地址。
  • git push origin <branch>：将本地分支推送到远程仓库。
  • git pull：从远程仓库拉取并合并代码到当前分支。
  • git fetch：从远程仓库获取最新代码，但不合并到当前分支。

4. 其他常用命令

  • git stash：将当前的修改保存到一个临时区域，以便切换分支或进行其他操作。git stash apply可以恢复保存的更改，git stash drop从临时区域中删除stash
  • git tag -a v1.0 -m "Initial release"：当前提交打一个标签
  • git reset <file>：将文件从暂存区移除，但保留修改。可以用 git reset --hard 还原到特定的提交版本，并且删除所有未提交的更改。
  • git rebase：将一个分支上的更改应用到另一个分支上，通常用于整理提交历史。
